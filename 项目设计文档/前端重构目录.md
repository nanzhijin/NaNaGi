# NaNaGi 前端重构目录

> 2026-06-16
> 设计骨架：NaNaGi 前端重构目录（信息架构 + 组件清单）
> 视觉规格：Marvis 侧边栏设计方案（色彩系统 + 动画策略 + 实施分期）
> 目标：双人格侧边栏布局 — 同一对话线程，不同面具表达

## 一、设计语言

**基调**：克制 + 微暖 + 可感知的人格化。骨架走 Marvis 式克制办公路线（灰白层级），细节保留 companion/worker 的性格差异。不做像素风（旧版特色保留在 `legacy/`）。

### 1.1 色彩系统

| 角色 | companion（小女儿, 默认） | worker（大女儿） | 用途 |
|------|-------------------------|-----------------|------|
| 背景主色 | `#faf9f5`（微暖奶油） | `#f6f7f9`（冷灰白） | 面板底色 |
| 表面色 | `#ffffff` | `#fcfcfd` | 消息气泡 / 卡片 |
| 主文字 | `#1e1e1e` | `#1a1a1c` | 正文 |
| 次文字 | `rgba(30,30,30,0.50)` | `rgba(26,26,28,0.48)` | 时间戳 / 辅助 |
| 品牌色 | `#c4815e`（暖陶土） | `#3b6e8f`（冷钢蓝） | 主按钮 / 选中态 / 气泡竖线 |
| 品牌色浅 | `rgba(196,129,94,0.08)` | `rgba(59,110,143,0.07)` | 悬停 / 标签底纹 |

**Persona 切换通过 CSS 变量联动**：
```css
:root {
  --brand: #c4815e;
  --brand-light: rgba(196, 129, 94, 0.08);
  --surface-bg: #faf9f5;
  transition: --brand 300ms ease, --brand-light 300ms ease, --surface-bg 300ms ease;
}
:root[data-persona="worker"] {
  --brand: #3b6e8f;
  --brand-light: rgba(59, 110, 143, 0.07);
  --surface-bg: #f6f7f9;
}
```

视觉意图：切换感是"同一间屋子换了灯光色温"，不是"跳到了另一个房间"。

### 1.2 字体与间距

- 中文：系统默认（PingFang SC / Microsoft YaHei）
- 代码：`"SF Mono", "Cascadia Code", monospace`
- 正文 14px / 1.6，标题 16px / 600，辅助 12px
- 4px 网格：紧凑 8px / 标准 12px / 区块 20px / 面板 16px

### 1.3 动画策略

> 原则：只动 `transform` + `opacity`，不动布局属性。纯 CSS，零依赖。

| 场景 | 实现 | 时长 | 缓动 |
|------|------|------|------|
| Persona 切换下划线 | `translateX()` CSS transition | 200ms | ease-out |
| 品牌色切换 | CSS 变量 `transition` | 300ms | ease |
| 消息入场 | `translateY(8px)` + `opacity` | 250ms | ease-out, stagger 30ms/条 |
| 技能广场面板 | `translateX()` 滑入 | 300ms | cubic-bezier(0.22,0.61,0.36,1) |
| 技能卡片入场 | stagger `scale(0.95)→1` + `opacity` | 200ms/张 | ease-out |
| 发送按钮 hover | `scale(1.05)` | 150ms | ease-out |
| 打字指示器 | 三点 `opacity` 交替闪烁 | 1.4s | ease-in-out |
| 唱片机折叠/展开 | `max-height` transition | 250ms | ease-out |

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

## 二、信息架构

```
NaNaGi 侧边栏 (默认 380px, 可拖拽 320~600px)
├── 顶栏 — Persona 切换 + 设置
│   ├── PersonaTabs: companion ┃ worker
│   └── ⚙ 设置 (账户 / 记忆管理 / 关于)
│
├── 消息流 (主区域, flex-grow, 可滚动)
│   ├── Agent 消息 (品牌色竖线 + 圆角气泡 + persona 标签)
│   ├── 用户消息 (右对齐, 深半阶背景)
│   ├── 工具调用卡片 (折叠式, 标题行 + 状态 + JSON 展开)
│   ├── 混元生图结果 (可拖拽, 下载按钮)
│   └── 唱片机识别结果 (SSE jukebox 事件内联)
│
├── 唱片机条 (可折叠, 36px 折叠 / 80px 展开)
│   └── DraggableImage → 识别动画 (scan-line + 粒子)
│
├── 技能广场入口 (品牌色浅底, 36px)
│   └── 点击 → 抽屉面板从右侧滑入 (宽度 60%, 最小 300px)
│       ├── 搜索 + 分类标签 (全部 │ 记忆 │ 生图 │ 搜索)
│       └── 2 列技能卡片网格 (图标 + 标题 + 描述 + [已启用])
│
├── 输入区域 (固定底部)
│   └── textarea (40~120px) + 发送按钮 (brand色 hover scale)
│
├── 📊 仪表盘 (仅 admin, 设置面板入口)
│   ├── 访客概览 + 情绪趋势 + 反馈汇总 + 成本统计
│   └── Observability Trace 时间线
│
├── 🔬 Worker 检索 (仅 worker 面具)
│   ├── 论文检索 / 教材查询 / 知识库浏览
│   └── 结果卡片 (质量评分 + 来源标注)
│
└── ⚙ 配置面板 (仅 admin)
    ├── PersonaMask 参数 (偏置滑块)
    ├── Self-Node 雷达图 (6 trait)
    ├── IWM 节点编辑器 (关系图 + 6维)
    └── 记忆衰减策略
```

## 三、Persona 切换的 UI 联动

| 行为 | 说明 |
|------|------|
| 消息流 | **不清空**，保持当前对话历史 |
| 品牌色 | CSS 变量过渡 300ms（暖陶土 ↔ 冷钢蓝） |
| 背景色 | 微调（微暖奶油 ↔ 冷灰白） |
| 气泡竖线 | 已发送消息的竖线色不变（历史记录），新消息用当前 persona 色 |
| 输入框 placeholder | companion：「和 NaNaGi 聊聊吧~」/ worker：「输入任务...」 |
| System Prompt | 下一个对话轮次切换 PersonaMask 偏置值（后端逻辑，前端无感） |
| 打字指示器 | 颜色随当前 persona |
